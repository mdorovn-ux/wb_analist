# WB analyst Landing

Одностраничный сайт для WB analyst. Лендинг показывает ценность программы, объясняет установку и ведет на ZIP-архив из GitHub Releases.

Текущая стабильная версия: `1.1.1`.

Стабильная ссылка скачивания:

```text
https://github.com/mdorovn-ux/wb_analist/releases/download/v1.1.1/WB-analyst-v1.1.1.zip
```

Релизные файлы публикуются в GitHub Releases и доступны пользователям по публичной ссылке.

## Локальный запуск

```bash
cd landing
npm start
```

Открыть:

```text
http://localhost:3000
```

## Деплой на Railway

1. Откройте Railway.
2. Нажмите `New Project`.
3. Выберите `Deploy from GitHub repo`.
4. Выберите репозиторий `mdorovn-ux/wb_analist`.
5. Укажите настройки сервиса:
   - Root Directory: `landing`
   - Start Command: `npm start`
6. Railway сам задаст переменную `PORT`.
7. После деплоя откройте публичный домен Railway.

## Как обновить версию на сайте

1. Соберите новую Windows-версию приложения.
2. Упакуйте папку приложения в ZIP, например:

```text
releases/WB-analyst-v1.2.0.zip
```

3. Обновите `latest.json` в корне проекта.
4. Опубликуйте GitHub Release:

```powershell
.\tools\create_github_release.ps1 -Tag "v1.2.0" -ArchivePath "releases/WB-analyst-v1.2.0.zip" -Title "WB analyst v1.2.0"
```

5. В `landing/public/index.html` обновите:
   - ссылку скачивания;
   - текст стабильной версии;
   - блок `Что входит в v1.1`, если менялся состав релиза.

6. Запустите локально `npm start` и проверьте страницу.

## Что должно быть на странице

- Понятная кнопка скачивания.
- Короткое объяснение, что программа делает.
- Блок установки: скачать ZIP, распаковать, запустить exe, вставить API-токены.
- Предупреждение, что программа для Windows.
- Предупреждение, что WB API лимиты могут влиять на время загрузки.
- Указание, что Python пользователю устанавливать не нужно.
- Указание, что API-токены пользователь вставляет сам.

## Проверка перед публикацией

```bash
cd landing
npm start
```

Проверить:

- `http://localhost:3000` открывается;
- кнопка скачивания ведет на GitHub Releases;
- на мобильной ширине текст не вылезает из блоков;
- блок установки и предупреждения видны без лишнего поиска;
- публичная ссылка скачивания открывает ZIP-архив из GitHub Releases.
