# v1.1 - Полировка релизной базы и доставки

## Цель версии

Сделать доставку программы аккуратной и понятной: пользователь скачивает актуальную версию с сайта/GitHub, видит номер версии в приложении, может открыть информацию о программе, а мы имеем понятный процесс сборки и публикации.

v1.1 не должна менять расчетную логику финансового отчета. Это сервисно-релизная версия: упаковка, версия, доставка, проверка, документация.

## Что должно стать лучше для пользователя

- Пользователь видит, какая версия программы у него установлена.
- На сайте и в приложении указана одна и та же версия.
- Скачивание идет через нормальный GitHub Release, а не через raw-файл из репозитория.
- В приложении есть окно `О программе`.
- В приложении есть ручная проверка новой версии или хотя бы понятная ссылка на страницу скачивания.
- Архив релиза имеет предсказуемое имя.
- Есть инструкция: как скачать, распаковать, активировать и запустить.

## Что должно стать лучше для разработки

- Версия хранится в одном месте в коде.
- Сборка exe использует версию из проекта.
- Landing и README легко обновлять под новую версию.
- Процесс релиза описан пошагово.
- `main` остается рабочей веткой разработки, `prod` - веткой проверенного релиза.
