# WB analyst Desktop

Desktop-приложение WB analyst помогает селлеру Wildberries сформировать управленческий отчет на основе WB Finance API: сводка, детализация, продажи, возвраты, товары, реклама, налоги, себестоимость, внешние расходы и Excel-выгрузка.

Текущая стабильная версия: `1.1.0`.

## Запуск из исходников

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

## Проверка

```bash
python -m compileall .
python -m pytest
```

## Сборка exe

```bash
python -m PyInstaller --noconfirm --clean --windowed --paths src --name "WB analyst v1.1.0" --add-data "src\wb_finance_analyst\resources;wb_finance_analyst\resources" main.py
```

После сборки проверьте:

- exe запускается без установленного Python;
- открываются разделы `Автоотчет`, `Себестоимость товаров`, `Инструкция`, `Логи`;
- открывается окно `О программе`;
- приложение закрывается без долгой задержки.

## Ручная активация

При первом запуске программа показывает код компьютера. Для генерации ключа запустите локальный калькулятор:

```bash
python tools/license_calculator.py
```

Вставьте код компьютера пользователя, нажмите `Сгенерировать ключ` и отправьте пользователю полученный ключ. Универсальный ключ в калькуляторе предназначен для внутренних тестов.

## Основной пользовательский сценарий

1. Откройте `Автоотчет`.
2. Нажмите `API Wildberries`.
3. Вставьте `Finance API token`. Для автоматической рекламы можно добавить `Promotion API token`.
4. Нажмите `Задать период` и выберите WB-период или ручной диапазон.
5. Выберите режим загрузки:
   - `Быстрая сводка` - только сводные данные WB;
   - `Полный отчет` - сводка, детализация, товары, прибыль по товарам и реклама.
6. При необходимости включите `Кэш`.
7. Нажмите `Загрузить отчет`.
8. Проверьте карточки, вкладку `Общая сводка`, предупреждения и сверку WB.
9. Если изменили себестоимость, расходы или налоги, нажмите `Обновить расчет`, чтобы пересчитать уже загруженные данные без нового запроса к WB.
10. Нажмите `Сохранить Excel`.

## Периоды

`WB-периоды` - программа получает список отчетов WB и позволяет выбрать готовый период. Это лучший вариант для точной сверки с кабинетом WB.

`Ручной диапазон` - пользователь задает даты самостоятельно. Подходит для быстрой проверки, но может не совпадать с официальным отчетным периодом WB.

## Себестоимость товаров

Раздел `Себестоимость товаров` позволяет добавлять, редактировать, удалять и сохранять себестоимость прямо в таблице. Основной ключ товара - артикул поставщика.

Поддерживаемые поля:

```text
Товар
Себестоимость за 1 шт
Упаковка за 1 шт
Комментарий
```

Если товар не найден в справочнике, себестоимость считается равной нулю, а в отчет добавляется предупреждение.

## Налоги и расходы

Налоги настраиваются через `Настроить налоги`. Режим налога с главного экрана убран, поэтому расчет берет настройки из отдельного окна.

Расходы периода задаются через `Расходы периода`. Они участвуют в чистой прибыли и могут быть пересчитаны кнопкой `Обновить расчет` без повторной загрузки WB API.

## Кэш

Кэш ускоряет повторные загрузки. Он привязан к периоду, типу отчета, источнику данных и безопасному отпечатку API-токена. Если пользователь сменит кабинет или токен, данные старого кабинета не должны подмешаться в новый отчет.

## Логи

Раздел `Логи` показывает журнал программы. Можно:

- фильтровать записи по дате и времени;
- скопировать логи в буфер;
- сохранить txt-файл;
- очистить лог.

API-токены в логи не записываются.

## Обновления

Окно `О программе` показывает версию и кнопку `Проверить обновление`. Проверка обращается к `latest.json` из GitHub Releases и не использует WB API.

Проверка обновлений читает `latest.json` из GitHub Releases. Репозиторий и релизные файлы должны оставаться публичными, чтобы пользователи могли проверять и скачивать новые версии без доступа к вашему аккаунту GitHub.
