# WB Finance Analyst 1.0: тест-план точности отчетов

## Цель

Довести модуль `Автоотчет` до максимальной точности перед сборкой `exe` версии 1.0.

Главная проверка: суммы в приложении и экспортном Excel должны сходиться с исходными отчетами Wildberries и с финансовой сводкой WB по выбранному периоду, типу отчета и настройкам.

## Область проверки 1.0

Проверяем только видимые для версии 1.0 разделы:

- `Автоотчет`
- `Себестоимость товаров`

Скрытые модули не проверяются как продуктовые функции 1.0:

- `Обзор`
- `Новый отчет`
- `Объединение отчетов`
- `Настройки`

## Основные источники данных

Для каждого тестового периода нужны:

1. Сырой детализированный отчет WB Excel.
2. Сводная информация WB из кабинета или из `sales-reports/list`.
3. Excel, сформированный WB Finance Analyst.
4. Настройки себестоимости товаров.
5. Настройки налогов.
6. Настройки расходов периода.
7. При необходимости данные WB Promotion API.

## Обязательные тестовые периоды

Минимальный набор:

| ID | Период | Тип отчета | Цель |
|---|---|---|---|
| P01 | 01.06.2026 - 01.06.2026 | Основной | Базовая дневная сверка |
| P02 | 02.06.2026 - 02.06.2026 | Основной | Повторяемость на другом дне |
| P03 | 04.05.2026 - 10.05.2026 | Основной | Недельный отчет |
| P04 | 04.05.2026 - 10.05.2026 | По выкупам | Проверка второго типа отчета |
| P05 | 04.05.2026 - 10.05.2026 | Оба | Проверка раздельного `report_kind` |
| P06 | Любой период без рекламы | Основной | Promotion API выключен |
| P07 | Период с рекламой | Основной | Promotion API включен |

Дополнительно перед релизом желательно проверить 1 полный месяц.

## Эталонные формулы сверки с WB

### Продажи и возвраты

Из сырого WB Excel:

- `Продажа WB` = сумма `Вайлдберриз реализовал Товар (Пр)` по строкам, где:
  - `Тип документа = Продажа`
  - `Обоснование для оплаты = Продажа`

- `Возвраты WB` = сумма `Вайлдберриз реализовал Товар (Пр)` по строкам, где:
  - `Тип документа = Возврат`
  - `Обоснование для оплаты = Возврат`

- `Продажа WB чистая` = `Продажа WB - Возвраты WB`

### К перечислению за товар

- `К перечислению по продажам` = сумма `К перечислению Продавцу за реализованный Товар` по продажам.
- `Возвраты к перечислению` = сумма `К перечислению Продавцу за реализованный Товар` по возвратам, визуально положительным числом.
- `К перечислению за товар` = `К перечислению по продажам - Возвраты к перечислению + компенсации`, если компенсации WB должны входить в товарную часть.

Для текущей версии компенсации фиксируем отдельно и сверяем, не попали ли они в чужие показатели.

### WB расходы и удержания

- `Логистика` = сумма `Услуги по доставке товара покупателю`.
- `Хранение` = сумма `Хранение`.
- `Удержания/выплаты` = сумма `Удержания`.
- `Штрафы` = сумма `Общая сумма штрафов`.
- `Операции при приемке` = сумма `Операции на приемке`.
- `Лояльность` = сумма расходов по программе лояльности, если WB отдает их отдельной колонкой.
- `Компенсация скидки по программе лояльности` должна учитываться отдельно и не должна случайно попадать в логистику или задержку перечисления.
- `Возмещение издержек по перевозке/складским операциям` должно быть отдельным показателем или явно исключенным показателем, но не должно попадать в `Разовое изменение срока перечисления денежных средств`.

### Итого к оплате WB

Эталон:

```text
Итого к оплате WB =
К перечислению за товар
- Логистика
- Хранение
- Удержания/выплаты
- Штрафы
- Операции при приемке
- Лояльность
```

Допуск: `0.01` руб. для Excel-сверки, `1.00` руб. для сверки с WB-кабинетом, если WB округляет сводку иначе.

## Проверка листов экспортного Excel

### Лист `Общая сводка`

Проверить:

- все обязательные показатели присутствуют;
- порядок показателей понятный;
- суммы совпадают с эталонными формулами;
- `Итого к оплате WB` совпадает со сводкой WB;
- нет двойного учета рекламы, налогов, расходов;
- `Маржинальность %` считается от `К перечислению за товар`.

### Лист `Управленческая прибыль`

Проверить:

- `Итого к оплате WB` берется из WB-сводки;
- `Себестоимость` равна сумме себестоимости проданных товаров с учетом чистого количества;
- `Упаковка` считается отдельно;
- `Реклама WB` равна расходу рекламы без итоговой строки;
- `Внешние расходы` равны настройкам периода;
- `УСН`, `НДС`, `Ручная сумма` соответствуют настройкам налогов;
- `Чистая прибыль` = `Итого к оплате WB - Себестоимость - Упаковка - Реклама WB - Внешние расходы - Налоги`;
- при изменении настроек и нажатии `Обновить расчет` суммы пересчитываются без удвоения.

### Лист `Продажи`

Проверить:

- попадают только строки продаж;
- служебные строки с нулевым количеством и нулевой суммой исключаются;
- количество = сумма `Кол-во`, а не количество строк;
- `WB перечисление` по итогу = `К перечислению по продажам`;
- возвраты не попадают в продажи;
- итоговая строка одна.

### Лист `Возвраты`

Проверить:

- попадают только строки возвратов;
- суммы показываются положительно;
- количество = сумма `Кол-во` по возвратам;
- итог = `Возвраты к перечислению`;
- итоговая строка одна.

### Лист `Товары`

Проверить:

- основная группировка идет по `Артикул поставщика / vendor_code`;
- название товара может храниться отдельно, но не является основным ключом;
- `Продано` = сумма количества продаж;
- `Возвращено` = сумма количества возвратов;
- `Чистое количество` = `Продано - Возвращено`;
- `К перечислению за товар` = продажи минус возвраты;
- нет пустых товаров с нулевыми значениями;
- нет строк `Итого`, если лист не должен содержать итог.

### Лист `Прибыль по товарам`

Проверить:

- группировка по артикулу поставщика;
- `К перечислению за товар` = продажи минус возвраты;
- себестоимость = `Чистое количество * себестоимость за 1 шт`;
- упаковка = `Чистое количество * упаковка за 1 шт`;
- внешние расходы распределяются только один раз;
- итоговая строка одна;
- итог по `К перечислению за товар` совпадает с общей сводкой;
- при повторном пересчете итог не удваивается.

### Лист `Реклама WB`

Проверить:

- расходы рекламы не падают в ошибку финансового отчета;
- сумма рекламы в сводке = сумма строк рекламы без итоговой строки;
- итоговая строка не попадает в повторные пересчеты;
- при отсутствии Promotion API отчет формируется с предупреждением.

### Лист `Налоги`

Проверить режимы:

- `Не учитывать`
- `УСН`
- `НДС`
- `УСН + НДС`
- `Ручная сумма`

Для каждого режима проверить:

- правильную базу;
- правильную ставку;
- правильную сумму;
- карточки на главном экране;
- экспорт в Excel.

### Лист `Предупреждения`

Проверить:

- предупреждения понятные;
- нет технического мусора;
- нет повторов одного и того же предупреждения десятки раз;
- отсутствие себестоимости показывается только для товаров с продажами.

## Проверка модуля `Себестоимость товаров`

### CRUD в таблице

Проверить:

1. Добавить товар.
2. Изменить артикул.
3. Изменить себестоимость.
4. Изменить упаковку.
5. Изменить комментарий.
6. Сохранить.
7. Закрыть и открыть приложение.
8. Убедиться, что данные сохранились.
9. Удалить строку.
10. Сохранить.
11. Убедиться, что строка удалена.

### Числовые форматы

Проверить ввод:

- `100`
- `100.50`
- `100,50`
- `1 000,50`
- пусто
- текст вместо числа

Ожидание: приложение не падает, некорректное число становится `0`.

### Влияние на отчет

Проверить:

1. Загрузить отчет.
2. Изменить себестоимость товара.
3. Нажать `Обновить расчет`.
4. Убедиться, что:
   - WB API не вызывается;
   - себестоимость изменилась;
   - чистая прибыль изменилась;
   - суммы не удвоились.

## Проверка пересчета без WB API

Сценарий:

1. Загрузить полный отчет из WB.
2. Записать значения:
   - `К перечислению за товар`
   - `Итого к оплате WB`
   - `Реклама WB`
   - `Себестоимость`
   - `Внешние расходы`
   - `Налоги`
   - `Чистая прибыль`
3. Изменить себестоимость.
4. Нажать `Обновить расчет`.
5. Проверить, что WB API и Promotion API не вызываются.
6. Проверить, что WB-суммы не изменились.
7. Проверить, что управленческие показатели изменились корректно.
8. Повторить 3 раза подряд.

Критерий: итоговые строки, реклама, расходы, налоги и прибыль не удваиваются.

## Проверка кэша

Сценарии:

### Использовать кэш

1. Загрузить отчет за период.
2. Повторить загрузку за тот же период.
3. Выбрать `Использовать кэш`.
4. Убедиться, что отчет открывается быстро.
5. Убедиться, что WB API не вызывается.

### Обновить из WB

1. Повторить загрузку за тот же период.
2. Выбрать `Обновить из WB`.
3. Убедиться, что программа выполняет запросы с учетом rate limit.

## Проверка rate limit и 429

Проверить:

- все Finance API запросы идут через единый `WBApiClient`;
- между Finance API запросами выдерживается пауза;
- при 429 читается `X-Ratelimit-Retry`;
- ожидание = `X-Ratelimit-Retry + 2 сек`;
- если заголовка нет, ожидание = `65 сек`;
- максимум 3 повтора;
- пользователь видит статус ожидания;
- ошибка показывается только после исчерпания повторов;
- кнопка загрузки заблокирована во время загрузки.

## Проверка UI

Проверить:

- видны только разделы `Автоотчет` и `Себестоимость товаров`;
- скрытые разделы не удалены из кода;
- кнопка называется `Загрузить отчёт`;
- налоговый режим отсутствует на главном экране;
- режим налога берется из окна `Настроить налоги`;
- поля сверху расположены в 2 ряда;
- статусная строка внизу не закрывает таблицы;
- карточки показывают правильные значения;
- при ручном налоге карточка `УСН` становится `Ручная сумма`, `НДС = 0`.

## Проверка Excel-экспорта

Для каждого тестового периода:

1. Сохранить Excel.
2. Открыть файл.
3. Проверить наличие листов:
   - `Обозначения`
   - `Общая сводка`
   - `Управленческая прибыль`
   - `Продажи`
   - `Возвраты`
   - `Товары`
   - `Прибыль по товарам`
   - `Реклама WB`
   - `Внешние расходы`
   - `Налоги`
   - `Предупреждения`
   - `Настройки отчёта`
4. Проверить скрытые технические листы `_DATA_*`.
5. Проверить, что технические листы не содержат итоговые строки, если они потом используются для пересчета.
6. Проверить, что файл открывается в Excel без предупреждений.

## Сборка exe версии 1.0

Перед сборкой обновить версии:

- `pyproject.toml`: `version = "1.0.0"`
- `src/wb_finance_analyst/domain/constants.py`: `APP_VERSION = "1.0.0"`
- `src/wb_finance_analyst/__init__.py`: `__version__ = "1.0.0"`

Установить сборщик:

```powershell
pip install pyinstaller
```

Сборка:

```powershell
pyinstaller --noconfirm --windowed --name "WB Finance Analyst" main.py
```

Проверить exe:

- запускается на чистой Windows-машине или отдельном профиле;
- открывается без консоли;
- создаёт папки настроек;
- сохраняет токены;
- загружает отчет;
- сохраняет Excel;
- закрывается без зависания.

## Критерии готовности 1.0

Версия 1.0 готова, если:

- все тестовые периоды сходятся с WB;
- расхождение по ключевым суммам не больше `0.01` руб. в Excel-сверке;
- нет критических ошибок при 429;
- нет удвоения итогов;
- UI показывает только нужные разделы;
- exe запускается и сохраняет отчет;
- пользователь может без помощи разработчика получить корректный управленческий отчет.
